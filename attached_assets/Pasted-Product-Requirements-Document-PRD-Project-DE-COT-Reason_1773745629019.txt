Product Requirements Document (PRD)
Project: DE-COT Reasoning Agent – Data-Efficient CoT Distillation Pipeline
Target Platform: Replit (full end-to-end implementation)
Version: 1.0 – Production-ready specification for Replit execution
Date: March 2026
Objective: Build the complete DE-COT system described in the paper using only Replit (no external servers, no local machines during development)
1. Project Goal
Implement an end-to-end reasoning distillation pipeline on Replit that achieves the core results reported in the paper:

Teacher: GPT-4 (via OpenAI API)
Student: Llama-2-7B
Technique: CoT generation → filtering → LoRA fine-tuning → implicit reasoning at inference
Target benchmarks: GSM8K ≥94%, CommonsenseQA ≥93%, AQuA ≥92%, overall reasoning accuracy ~96–97%
Inference style: no explicit CoT at runtime (internalized during training)
Latency goal: ~0.6–0.9 s per reasoning question on Replit GPU

2. High-Level Pipeline (must exist as separate runnable scripts)
textphase 0    →  environment + dependencies + secrets

phase 5    →  evaluation & metric collection (paper-style tables)
phase 6    →  merge & deploy as FastAPI inference service

6. Acceptance Criteria (what “done” looks like)

All six main scripts can be executed independently with appropriate arguments/flags.
Running the full pipeline (01→06) produces:
a LoRA/QLoRA adapter that loads successfully with peft
evaluation numbers within ~1–3% of paper reported values on at least GSM8K & AQuA

FastAPI server starts on Replit and correctly answers 5–10 sample reasoning questions with < 2 s latency (cold start acceptable)
Total OpenAI spend is logged and visible in the final report
README contains clear instructions how to:
install dependencies
set secrets
run each phase
resume interrupted training
deploy the API persistently
